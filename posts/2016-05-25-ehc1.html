Intro to Haskell and functional programming, and lambda calculus.

Functional Programming

Haskell is a purely functional programming language. It’s a generic language, meaning, it’s not domain specific.
It’s functional, as functions are treated as first class values.
It’s pure, as it’s based purely on the concept of the lambda calculus, without breaking it, to allow for IO.

First class values.

So, what does first class value mean?
First of all, what is a value?
A programming language defines a way to compute “values”, and work with them, being numbers, strings, dates. In case of numbers for example, we can add them, and strings print them to screen. So a first class value, is just something that we can work with directly in the language, and in functional ones, it means we can also consider functions like this.

Functions

The most general definition of a function is a mapping between two sets. A set is just a collection of elements, provided that no element is present multiple times.

Examples of a set would be

A = {1,2,3}
B = {z,y,x}
C = {0}
D = {} (empty set)

but not

E = {1,1,2}
F = {z,y,z}

For example, a function f mapping from set A to B, would be:

f(1) = z
f(2) = y
f(3) = x

Or g:

g(1) = z
g(2) = z
g(3) = z

Or h:

h(1) = z

h is also called a partial function, as it doesn’t define the mapping for each element from A to B, while f and g, that do, are called total functions.

Another way of defining functions is by defining an expression, like so

f(x) = x + 1

And that “x” is an argument to the function, while “x+1” is the body of the function.
So, when that gets applied for each member of the set A, it becomes:

f(1) = 2
f(2) = 3
f(3) = 4

or also

f(1) = 1 + 1
f(2) = 2 + 1
f(3) = 3 + 1

And the two definitions are not distinguishable, as they have the same behaviour.
And this brings us to the main point of this lesson.

Lambda Calculus.

From wikipedia: Lambda calculus (also written as λ-calculus) is a formal system in mathematical logic for expressing computation based on function abstraction and application using variable binding and substitution.

So uh, that’s a lot. But basically it’s just formal way of defining these computations. We’ll see those abstractions and applications in the following paragraphs, don’t worry.
